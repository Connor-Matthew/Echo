# Echo Chat Home High Fidelity Atelier Design

日期：2026-03-20
状态：已确认，待进入实现计划

## 1. 背景

本次改版目标是将 Echo 当前首页和聊天主页，重排为更接近参考图的高保真工作台界面。目标不是抽象借鉴参考图的配色，而是在保留 Echo 现有聊天、附件、Skills、MCP、模型切换等核心能力的前提下，让首页和聊天主界面在第一眼上明显接近参考图的结构、节奏和氛围。

当前代码库已经有一轮 `atelier-lite` 尝试，但整体仍更像在原有聊天壳层上做视觉精修，没有完全建立参考图中的四层结构：

- 左侧轻导航工作台
- 顶部横向品牌导航
- 中央舞台式对话区
- 底部超圆悬浮输入栏

本次设计的结论是：只改首页和聊天页，不扩散到设置页和 Agent 页；并且采用高保真重排路线，而不是继续做中度贴近的外观微调。

## 2. 范围

本次只覆盖 Echo 的首页和聊天主界面。

包含：

- 聊天首页空状态
- 有会话时的聊天主界面
- 左侧导航栏
- 顶部品牌导航与搜索区
- 消息舞台和附件呈现
- 底部固定输入区

不包含：

- Settings 全量视觉重做
- Agent 页重做
- 新业务能力开发
- 真实全局搜索能力补全
- 新增永久右侧上下文栏

## 3. 设计目标

### 3.1 视觉目标

- 让首页第一眼明显贴近参考图，而不是仍然像通用桌面聊天工具
- 保留暖白纸感、陶土橙强调色、编辑感留白和柔和阴影
- 通过布局重排建立“工作台”感，而不是只靠颜色和圆角

### 3.2 产品目标

- 保留 Echo 现有聊天交互主轴，不更改发送、流式、附件、Skills、MCP 等核心逻辑
- 让空状态和真实聊天状态都保持统一，而不是只做一个静态 landing mock
- 维持当前 React 组件边界，优先做 presentational 层改造

### 3.3 成功标准

- 打开聊天首页时，主观感受显著接近参考图
- 进入真实对话后，不会从“设计页”退化回旧聊天页
- 现有功能不回归，尤其是附件、流式、模型切换与技能入口

## 4. 总体结构

### 4.1 页面骨架

聊天主页采用四层结构：

1. 左侧固定侧栏
2. 顶部横向品牌导航
3. 中央对话舞台
4. 底部固定 hero composer

页面不新增复杂多栏仪表盘。参考图中右侧的文件卡、图像块等视觉重点，应通过现有消息流和附件能力吸收，而不是额外发明新的并列主区。

### 4.2 布局节奏

- 左栏宽度保持约 `260-280px`
- 中央阅读舞台控制在 `920-980px` 最大宽度
- 真正消息阅读列保持在 `700-760px` 的舒适范围
- 底部输入区固定于中轴下方，并与中央阅读列对齐

## 5. 核心界面设计

### 5.1 左侧侧栏

侧栏将更接近参考图中的轻导航工作台，而不是当前更偏功能面板的表现。

保留和重排的元素：

- 顶部品牌区：主品牌使用 `Echo`，副标题采用轻量产品描述
- 主按钮：`New Chat`
- 次级入口：`History`、`Prompts`、`Library`
- 底部入口：`Settings`
- 底部卡位：预留状态卡或升级卡的视觉结构

交互原则：

- 当前激活项以轻底色或细强调线表示
- 会话列表继续存在，但视觉权重下降为“索引”
- 不引入与当前业务无关的营销信息

### 5.2 顶部导航

顶部导航高保真参考图中的结构，但内容映射回 Echo 当前能力。

建议结构：

- 左侧主标题：`Echo Silk`
- 导航项：`Chat`、`Prompts`、`Library`
- 右侧：搜索框、头像或个人入口占位

说明：

- `Echo Silk` 作为当前聊天主页标题使用，表达本次主页主题，不替代产品名 `Echo`
- `Chat` 对应当前聊天主视图
- `Prompts` 使用现有技能或 prompt 语义承接
- `Library` 第一版允许为轻量入口，不承诺完整资料库能力
- 搜索框第一版优先提供视觉和交互壳层，不强行补完整全局搜索业务

### 5.3 中央对话舞台

中央区域从“消息列表容器”提升为“舞台式对话区”。

空状态时：

- 显示更有品牌感的欢迎文案
- 留白更大，视线自然落到输入栏
- 保留轻量引导，不堆砌功能说明

有消息时：

- 用户消息显示为右侧暖橙短气泡
- 助手消息显示为更宽的奶油色编辑感卡片
- 附件、结构化结果、引用块继续沿用消息流模型，但视觉上更接近参考图中的嵌入式文件卡或成果卡

### 5.4 底部输入区

输入区是本次改版最关键的交互锚点，需尽量贴近参考图。

要求：

- 采用长条超圆角悬浮输入栏
- 左侧保留加号和附件入口
- 右侧保留发送按钮与未来扩展位
- 中间文本输入区域保持低干扰、可长时间阅读和输入

实现原则：

- 外观高保真重绘
- 内部逻辑继续复用现有 `Composer` 组件能力
- 不拆掉当前模型、MCP、Skills、上下文窗口等已有行为，只调整在 minimal/home 模式下的呈现方式

## 6. 视觉系统

### 6.1 气质关键词

- editorial
- atelier
- warm paper
- calm
- soft premium

### 6.2 色彩

- 背景：暖白、米白、浅奶油
- 强调色：低饱和陶土橙
- 正文：深暖灰，不使用纯黑
- 次要文字：暖灰中性色

原则：

- 大面积使用浅暖底色建立层次
- 强调色仅用于关键交互和用户气泡
- 减少传统硬边框感，更依赖层差、阴影和留白

### 6.3 形态与阴影

- 大壳层圆角：`24-30px`
- 卡片圆角：`20-26px`
- 输入栏圆角：更接近胶囊状超圆角
- 阴影偏大、偏软、偏低对比

### 6.4 排版

- 顶部标题采用更强的编辑感字重与字距
- 正文消息优先可读性，不为了造型压缩行高
- 小字对比度必须足够，不追求“很淡很高级”的不可读效果

## 8. 高保真参考与产品映射边界

### 8.1 高保真参考内容

以下部分应尽量贴近参考图：

- 左侧导航和品牌区整体节奏
- 顶部横向导航结构
- 右上搜索框与个人入口占位
- 中央暖橙用户气泡和奶油色助手卡片
- 底部长条超圆输入栏

### 8.2 需要用 Echo 现有能力替代的部分

- `Prompts` 使用现有 Skills 或 prompt 语义承接
- `Library` 先作为轻量入口，不构造完整新模块
- 文件卡和附件卡直接复用现有消息附件模型
- 底部输入区保留现有业务能力，不为贴图而发明新逻辑

## 9. 非目标与风险

非目标：

- 不把整站所有页面同时改版
- 不为了模仿参考图而加入静态假内容
- 不引入永久右侧 dock

主要风险：

- 顶部导航和现有聊天 header 的职责重叠
- Sidebar 既要像参考图，又要承接现有真实会话列表，信息密度需要平衡
- Hero composer 外观重排后，不能影响附件、MCP、Skills、上下文窗口的可达性

应对策略：

- 第一阶段优先完成壳层和主视觉结构
- 导航项允许第一版部分为轻入口或占位交互
- 所有行为逻辑尽量保持原样，仅修改视觉层和排版层

## 10. 实施原则

- 先改壳层，再改消息与输入区
- 先做聊天主页和真实聊天状态统一，再考虑额外点缀
- 所有设计选择以“更像参考图且不破坏 Echo 现有使用流”为准

最终结论：

本次首页和聊天页改版采用“高保真 Atelier 重排”方案。实现上不重写业务逻辑，只围绕现有 React 组件骨架做版式、视觉层级和外壳重构，使 Echo 的聊天主页在视觉上尽量接近参考图，同时保持真实可用。
