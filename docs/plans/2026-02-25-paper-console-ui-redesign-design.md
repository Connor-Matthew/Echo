# Paper Console UI 重构设计

日期：2026-02-25  
状态：已确认，进入实现

## 1. 重构目标

本次 UI 重构目标不是小修补，而是把聊天界面重建为更克制、更有编辑感的 `Paper Console` 风格。

已确认决策：

- 风格方向：`Paper Console`（暖灰纸感、低对比、大留白、细描边）
- 首页文案：`仅在新会话（0 消息）显示`
- 侧栏策略：`B 平衡型`（极简外观 + Workspace 可展开分组）
- 输入区策略：`A 悬浮底部舱`（始终贴底悬浮）

成功标准：

- 首屏呈现干净稳定的留白结构，不再是“卡片堆叠”视觉。
- 会话阅读效率高，用户消息与助手消息层级清晰。
- 输入舱成为唯一强视觉锚点，交互稳定可预期。
- 不改动现有核心业务逻辑（会话、流式、附件、MCP、Skills）。

## 2. 信息架构与布局

采用三段式结构：

1. 左栏：`272px`（可折叠为 `72px`）  
2. 中间：对话舞台（内容宽度上限 `960px`，阅读列 `720-760px`）  
3. 底部：悬浮输入舱（距底 `24px`，高度约 `112-136px`）

侧栏内容优先级：

- 常驻：New Chat、Search、会话列表
- 收纳：Agent / Settings / Tools 放入 Workspace 分组

主区域：

- 新会话无消息时显示欢迎文案
- 有消息后展示标准对话流
- 输入舱固定贴底，不随内容滚动

## 3. 视觉系统

### 3.1 色彩

- 背景：`Warm Paper`（接近 `#F7F5F2`）
- 侧栏：更深一阶暖灰（接近 `#EFECE8`）
- 正文：`#2F2D2A`
- 次级文本：`#7E7872`
- 边框：`#D9D3CC`
- 强调色：`Terracotta`（接近 `#C86E45`）

强调色只用于：

- 发送按钮
- 激活态细竖条
- 关键 CTA

### 3.2 形态

- 圆角三档：`10px / 16px / 24px`
- 阴影：单层软阴影，弱存在感
- 边框：以 1px 细描边为主，不使用厚重描边

### 3.3 字体

- 正文：`Noto Sans SC`
- 标题/欢迎语：`Noto Serif SC`（轻编辑感）
- 代码：保留等宽字体

## 4. 组件策略

### 4.1 Sidebar

- 采用平衡型极简导航
- 会话激活态使用左侧 `2px` 竖条，不用整块强高亮
- 一级操作按钮保持低视觉噪声

### 4.2 ChatView

- 助手消息偏“文本块”呈现（弱气泡）
- 用户消息右对齐浅暖灰气泡（16px 圆角）
- 消息间距 `16-20px`，段落行高 `1.7`

### 4.3 Composer

- 输入舱为主视觉锚点
- 附件、模型、发送在同一操作层
- 聚焦时轻微抬升与描边强化

### 4.4 Attachment Tray

- 与输入舱同风格描边和底色
- 轻卡片样式，避免“工具面板感”

## 5. 动效与反馈

动效统一 `160-220ms ease-out`，只保留三类：

1. 侧栏折叠/展开
2. 输入舱聚焦抬升
3. 消息渐入（轻上移 + 淡入）

避免：

- 弹跳
- 大幅缩放
- 高频闪烁

## 6. 实施顺序

1. 全局设计 token（颜色、字体、圆角、阴影）  
2. App 壳层布局重排（Paper Console 框架）  
3. Sidebar / ChatView / Composer / AttachmentTray 组件重绘  
4. 交互动效与细节收口  
5. typecheck / build 回归验证

- 会话切换与滚动行为正常
- 拖拽上传、流式输出、错误提示不回归
- Agent/Settings 入口仍可达，状态可感知
