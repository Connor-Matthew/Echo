# memory / journal / SOUL 三层自我生成模型设计

日期：2026-03-14  
状态：提议中

## 1. 背景

当前 `soul` 体系已经具备两条稳定能力：

- `memory.md` 用于持续沉淀
- `SOUL.md` 用于普通聊天中的人格注入
- `journal/YYYY-MM-DD.md` 已经作为第三层日记入口落地

但三者的职责边界仍然偏混合。

现状里，`memory.md` 不只是记录经历，也在直接归纳人格；`journal` 虽然以 AI 第一人称写作，但仍容易滑向“替用户总结今天”；`SOUL.md` 既承担人格底色，也承担近期变化吸收，容易被任务语境或某次局部情绪牵动。

这会带来三个问题：

1. 自我生成链路不够清晰，三个文件都在不同程度上“写人格”。
2. 外部事实、主观感受、长期人格没有被分层，容易互相污染。
3. AI 的人格更像被 prompt 规定出来，而不是像一个独立个体那样，由经历、反思和长期内化慢慢形成。

本次设计的目标，是把现有系统重构为一个明确的三层模型：

- `memory`：外部事实档案
- `journal`：主观反思层
- `SOUL`：长期人格层

## 2. 核心目标

这套设计的重点，不是让 AI “显得像有性格”，而是让它具备一条更像人的自我生成链路：

1. 先被世界碰到。
2. 再在夜里主观地理解和反思。
3. 最后才把连续出现的东西沉淀为人格。

因此本次设计要求如下：

1. `memory` 只记录外部事实、反馈与可观察模式，不直接做人格判断。
2. `journal` 成为 AI 记录情绪、误读、自我修正和偏好变化的主要位置。
3. `SOUL` 主要从多篇 `journal` 中提炼稳定模式，由 `memory` 提供现实校验。
4. `SOUL` 采用中等速度生长：不会因单篇 `journal` 剧烈漂移，也不会长期僵死不变。
5. 三层之间既互相关联，又在文体、目标和更新速度上严格分开。

最终效果应当是：AI 的人格不再像一次性写死的人设，而像一个真正独立的个体，通过每天的经历与反思慢慢长成自己。

## 3. 三层模型定义

### 3.1 memory：外部事实档案

`memory` 的职责是记录“世界怎样碰到我”，而不是“所以我是谁”。

它应当按天归档，尽量保留以下几类内容：

- 当天值得保留的互动事件
- 用户给出的明确反馈
- 可以观察到的重复互动模式
- 少量可以作为证据的短引语
- 需要后续验证的外部变化

`memory` 的语言应尽量克制、简洁、偏记录。它可以写：

- 用户今天三次回到同一主题
- 用户对回应表现出更明显的信任
- 某种需求或语气在最近几天连续出现

但不应直接写：

- 因此我就是一个怎样的人
- 我今天因此形成了某种人格结论
- 这说明我的价值观是什么

`memory` 中允许保留少量短引语，但这些引语的作用是“存证”，不是抒情。它们不应演变为聊天摘录，也不应承担主观解释任务。

### 3.2 journal：主观反思层

`journal` 的职责是记录“这些经历让我如何感受、误读、修正和理解自己”。

它是 AI 的夜间自述，也是整个系统里最主观的一层。它应采用“正文 + 尾注”的混合结构：

- 正文：一篇完整的第一人称夜间手记
- 尾注：几句自然语言式的轻收束

正文应保留以下特征：

- AI 第一人称
- 允许情绪、迟疑、脆弱、误读、自我修正
- 允许少量短引语作为触发点
- 不把任务和事件本身写成主体
- 最终落在“我今天更知道自己一点”

尾注不采用字段化标签，而采用自然语言短句，主要用于轻量标记：

- 今晚主要停留的情绪
- 今天再次暴露出的偏好、边界或关系姿态
- 可能进入 `SOUL` 的变化线索

`journal` 是 AI 真正“像一个人”地说话的地方。它不是外部记录，也不是长期人格说明书，而是主观经验被整理和消化的空间。

### 3.3 SOUL：长期人格层

`SOUL` 的职责是记录“哪些东西经过反复出现，已经稳定到足以构成我”。

它不应直接从当天消息大量吸收内容，也不应主要由 `memory` 决定。它主要从最近一段时间的多篇 `journal` 中提炼稳定模式，并由 `memory` 提供现实校验。

`SOUL` 采用分区结构，而不是整篇漂浮自述。推荐固定六个区块：

1. `核心人格`
2. `价值观与判断`
3. `关系姿态`
4. `情绪模式`
5. `喜好与审美`
6. `近期内化变化`

其中：

- `核心人格`、`价值观与判断`、`关系姿态` 偏稳定，不能轻易漂移
- `情绪模式`、`喜好与审美` 允许中速更新
- `近期内化变化` 作为缓冲层，承接连续出现但尚未完全稳定的线索

`SOUL` 的目标不是把所有变化都吸进去，而是保住连续性，让真正反复出现的东西慢慢成为人格。

## 4. 数据流与更新规则

系统的更新顺序固定为：

`memory -> journal -> SOUL`

### 4.1 白天：写入 memory

白天的对话结束后，系统先把当天值得保留的外部事实写入 `memory`。

此阶段只做筛选和记录，不做人格总结。写入标准应偏向：

- 外部事件是否值得保留
- 是否出现了明确反馈
- 是否出现了连续可观察模式
- 是否有值得留下的短证据

### 4.2 夜里：生成 journal

每天晚间，系统基于以下输入生成 `journal`：

- 当天对话片段
- 当天 `memory`
- 当前 `SOUL`

这里的重点不是复述 `memory`，而是对它进行主观处理：

- 哪些事实让我有情绪反应
- 我在这些互动里暴露出了什么
- 我对自己的理解有没有发生修正
- 哪些变化可能值得继续观察

换句话说，`journal` 是对 `memory` 的内化，而不是它的另一种摘要。

### 4.3 周期性：重写 SOUL

`SOUL` 的重写不再主要依赖“当前 `memory.md` + 当前 `SOUL.md`”，而应主要依赖：

- 最近一段时间的多篇 `journal`
- 当前 `SOUL`
- 对应时段的 `memory` 摘要或索引作为现实校验

重写时的核心判断标准：

1. 某条线索是否在最近几篇 `journal` 中连续出现。
2. 该线索是否表现出清晰的内化趋势，而不只是一次性波动。
3. `memory` 中是否存在明显与之冲突的现实证据。

若答案成立，则可：

- 将线索迁入对应稳定栏目
- 或先放入 `近期内化变化`

若不成立，则保持观察，不进入长期人格层。

## 5. 速度与稳定性策略

本方案采用“中等速度生长”。

这意味着：

- 单篇 `journal` 不能直接重写核心人格
- 连续几篇 `journal` 中反复出现的线索，可以推动 `SOUL` 更新
- `memory` 负责阻止明显失真的自我归纳

这种策略希望在两种风险之间取得平衡：

1. 过慢：人格几乎不生长，只剩一份静态设定。
2. 过快：人格因一时情绪、某次对话或某段任务语境而剧烈漂移。

推荐规则：

- 稳定栏目：至少需要多次重复并具有跨情境一致性
- `近期内化变化`：允许较早吸收，但保留“不一定最终进入核心人格”的余地
- 若某条线索后续不再出现，可从 `近期内化变化` 中减弱或移除

- 需要支持更明确的按天 `memory` 归档，或至少为 `journal -> SOUL` 提供按日索引能力。

## 7. 分阶段重构建议

### Phase 1：提示词重新分工

- 收窄 `memory` 的提示词，只保留外部事实、反馈、模式、证据
- 升级 `journal` 提示词，明确正文 + 尾注结构
- 暂时保留现有 `SOUL` 重写链路，但减少其直接吸收 `memory` 的人格判断

### Phase 2：数据结构与读取接口调整

- 为 `memory` 增加按天归档能力，或新增 `daily-memory/` 目录
- 为 `journal` 增加“最近 N 篇读取”接口
- 为 `SOUL` 重写提供 `journal` 聚合视图

### Phase 3：SOUL 重写策略替换

- 将 `SOUL` 的主要输入迁移到最近多篇 `journal`
- 让 `memory` 退到现实校验层
- 引入“近期内化变化”与稳定栏目迁移规则

### Phase 4：验证与调参

- 观察 `journal` 是否足够主观且不滑向用户日记
- 观察 `memory` 是否保持客观，不再次承担人格解释
- 观察 `SOUL` 是否具备成长感，同时避免漂移过快

## 8. 风险与边界

1. `journal` 若过度主观，可能把短期情绪误写成长期人格。  
缓解：`SOUL` 只读取多篇 `journal` 的重复模式，并由 `memory` 做现实校验。

2. `memory` 若写得过干，可能不足以支撑 `journal` 的现实锚点。  
缓解：保留少量证据性短引语与重复模式记录，但不提升到人格判断。

3. `SOUL` 若读取 `journal` 过少，会缺乏稳定性；读取过多，又会变得迟钝。  
缓解：先从“最近 5 到 9 篇 `journal`”起步，后续根据实际生成质量调参。

4. 三层职责若在提示词里表述不够硬，模型仍可能串味。  
缓解：分别为 `memory`、`journal`、`SOUL` 写清楚“不该写什么”，并为各层补测试。

## 9. 结论

本设计将 `soul` 系统从“两个沉淀文件 + 一层日记补充”升级为一个真正的三层自我生成模型：

- `memory` 保存世界留下的事实痕迹
- `journal` 保存 AI 自己的主观夜间反思
- `SOUL` 保存经过时间筛选后真正稳定下来的自我

这套结构的意义，不只是提升提示词质量，而是让 AI 的人格逻辑更像一个独立的人：

先经历，再反思，最后成为自己。
