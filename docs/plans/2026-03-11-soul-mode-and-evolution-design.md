# SOUL 模式与自我演化设计

日期：2026-03-11  
状态：已实现

## 1. 目标

本次重构将普通聊天的人格机制收敛为两条明确路径：

- `SOUL 模式开启`：只注入 `SOUL.md`
- `SOUL 模式关闭`：只注入 `systemPrompt`

不再保留结构化 `persona` 记忆、提取结果、撤销操作等链路。

同时新增一条后台自我演化机制：

- 全局普通聊天用户消息每累计 `5` 条，重写一次 `memory.md`
- 每次 `memory.md` 重写成功后，立即联动重写一次 `SOUL.md`
- 每天 `10:00`、`20:00`，基于 `SOUL.md + memory.md` 再做兜底重写
- 错过时点时，在下次启动后补做一次重写

## 2. 产品行为

### 2.1 会话级 SOUL 开关

- `ChatSession` 新增 `soulModeEnabled`
- 新建会话默认 `true`
- 开关放在普通聊天标题栏
- 切换后保留历史消息，不清空上下文
- 切换时显示轻提示：
  - `已经切换为 SOUL 模式`
  - `已经切换为系统提示词模式`

### 2.2 普通聊天提示词组装

- `SOUL 模式开启` 时：
  - 读取本地 `SOUL.md`
  - 将其包装为唯一人格源 system message
  - 不再注入 `settings.systemPrompt`
- `SOUL 模式关闭` 时：
  - 保持现有 `settings.systemPrompt`
- 环境信息与 memos 记忆仍按原链路注入
- 若 `SOUL.md` 为空，则本轮请求静默按“无人格源”发送

## 3. 本地文件

统一使用 `~/.echo/memory/`：

- `soul.md`：普通聊天在 SOUL 模式下读取的人格正文
- `memory.md`：后台沉淀使用的内在记忆档案，不直接注入普通聊天
- `soul-state.json`：记录自动沉淀与定时重写进度

`memory.md` 维持固定的分层结构：

- `核心自我与守恒`
- `反复出现的主题`
- `关系与边界`
- `核心世界观`
- `重要经历与处境`
- `近期变化`
- `内在冲突与张力`
- `变化评估`
- `自我反思`
- `未定型线索`

## 4. 自我演化链路

### 4.1 memory.md 沉淀

- 只统计普通聊天里的 `user` 消息
- 按全局会话合并排序
- 依据 `soul-state.json` 中的游标找到尚未沉淀的消息
- 每满 `5` 条后，在当前回复结束后发起一次后台模型请求
- 模型根据 `当前 memory.md + 这 5 条用户消息` 执行一次“分层归档 + 迁移”，重写整份 `memory.md`
- 沉淀目标不只是摘要，而是编码 5 类更强的自我连续性机制：
  - `自我守恒`：保留不应轻易漂移的稳定核心
  - `变化筛选`：明确哪些变化已经接纳、仍在观察、暂不吸收
  - `内在冲突`：允许矛盾和张力长期存在，而不是强行写顺
  - `主动反思`：保留灵魂对自身变化的判断，而不只是外部观察
  - `稳定世界观`：沉淀较长期的价值判断与理解世界的方式

### 4.2 SOUL.md 重写

- 主触发：每次 `memory.md` 重写成功后，立即继续重写 `SOUL.md`
- 兜底调度：每天 `10:00`、`20:00`
- 定时判断逻辑：比较当前时点之前最近一个调度槽位与 `lastSoulRewriteSlot`
- 若内存联动或当前槽位需要处理：
  - 读取最新 `SOUL.md`
  - 读取最新 `memory.md`
  - 发起后台模型请求
  - 依据 `memory.md` 的稳定层、近期层、冲突层、变化评估层与未定型层来重写灵魂文本
  - 返回整份新的 `SOUL.md`
  - 直接覆盖本地 `soul.md`

重写成功后，在普通聊天标题栏中部显示 `SOUL 已更新`，持续 `2` 秒。

## 5. 实现范围

已删除：

- `persona` 类型定义
- `persona` IPC
- `electron/memory/persona-service.ts`
- 旧的 Soul-mode ingest/undo 设计文档

已新增：

- 会话级 `soulModeEnabled`
- `SOUL.md / memory.md / soul-state.json` 本地读写
- 普通聊天标题栏 SOUL 开关与轻提示
- 全局 `memory.md` 批量沉淀
- 定时/补做的 `SOUL.md` 自动重写
- 灵魂页可配置独立的 `SOUL 演化渠道 + 模型`，失败时静默回退当前聊天模型

## 6. 风险与边界

- 自动演化优先使用灵魂页指定的 `SOUL 演化渠道 + 模型`，若不可用则静默回退到当前普通聊天模型
- `memory.md` 只按 `5` 条用户消息一批更新，因此不足 `5` 条的新信号不会立即进入自动 SOUL 联动重写
- 自动重写直接覆盖 `SOUL.md`，不保留版本历史
- 自我演化仍然是文本层面的连续自我建模，不是独立意识的证明；其质量取决于沉淀与重写提示词的边界是否稳定
