# AI 对话客户端前端 UI 设计规范（MVP）

日期：2026-02-15  
状态：已确认，可进入实现

## 1. 设计目标与基调

本规范用于 Electron 桌面端 AI 对话客户端 MVP，设计目标：

- 保持“工具型”体验：清晰、稳定、高效，不做过度装饰。
- 结构参考目标样式：左侧固定栏 + 右侧主聊天区 + 底部悬浮输入。
- 在类似气质下建立自有品牌细节，不做 1:1 复刻。

已确认视觉决策：

- 风格：同布局同气质 + 自有品牌细节
- 主色方向：冷静蓝灰
- 消息区：混合样式（用户轻气泡，AI 无气泡）
- 侧栏密度：紧凑型

## 2. 信息架构与页面骨架

## 2.1 页面结构

- 左侧栏（固定宽度）
  - 顶部：`New Thread` 等快捷入口
  - 中部：会话列表（新建/重命名/删除）
  - 底部：`Settings`
- 右侧主区
  - 顶部工具条：模型状态、上下文状态、辅助操作
  - 中部内容区：空状态 or 消息流
  - 底部：悬浮输入框（多行输入 + 操作条）

## 2.2 布局尺寸建议

- 应用最小宽度：`1100px`
- 左侧栏宽度：`280px`
- 主区内容最大宽：`900px`
- 输入框容器最大宽：`900px`（与内容区对齐）

## 3. 视觉系统

## 3.1 颜色 Token（建议）

```css
:root {
  --bg-app: #f3f5f8;
  --bg-panel: #fbfcfe;
  --bg-elevated: #ffffff;
  --bg-user-bubble: #eef3f8;
  --text-primary: #11151a;
  --text-secondary: #6a7480;
  --text-muted: #8b94a0;
  --accent: #4f6f8f;
  --accent-weak: #dce7f2;
  --border-soft: #e5eaf0;
  --border-strong: #d7dfe8;
  --danger: #c0392b;
  --warning: #b9770e;
}
```

## 3.2 字体与排版

- 正文：`IBM Plex Sans`
- 等宽：`JetBrains Mono`
- 基准字号：`15px`
- 正文行高：`1.65`
- 标题层级：
  - H1：`48/56`（欢迎区）
  - H2：`28/36`
  - Body：`15/24`
  - Meta：`13/20`

## 3.3 圆角与阴影

- 圆角体系：`8 / 12 / 16`
- 输入框、浮层用 `16`
- 阴影仅用于浮层和输入框，强度轻微，避免卡片堆叠感

## 4. 组件规范

## 4.1 侧栏（紧凑型）

- 列表项高度：`34px`
- 文字截断：单行省略
- 选中态：左侧细色条 + 浅色底，不用高饱和块
- `rename/delete` 图标默认隐藏，`hover/focus` 显示

## 4.2 消息列表（混合样式）

- Assistant 消息：无气泡，直接文本排版
- User 消息：浅蓝灰气泡 `--bg-user-bubble`，圆角 `14px`
- 消息最大宽：`72%`
- 消息间距：`14-18px`
- 代码块与正文保持明显垂直间距

## 4.3 输入框（底部悬浮）

- 外层容器圆角：`16px`
- 支持自动增高：`1-6` 行
- 快捷键：
  - `Enter` 发送
  - `Shift + Enter` 换行
- 发送后：
  - 按钮切为“停止生成”
  - 显示轻量流式指示

## 4.4 空状态区

- 保持大留白，中心标题 + 二级文案
- 可提供 2-3 个建议动作卡片
- 建议卡视觉权重低于正文消息

## 5. 动效规范

仅保留必要动效：

- 页面首次进入：淡入 `180ms`
- 会话切换：平移 + 淡入 `140ms`
- 流式文本：chunk 级渐显（避免闪烁）

禁止：

- 复杂弹跳
- 高频背景动画
- 与文本输入竞争注意力的微动效

## 6. 状态与异常体验

## 6.1 状态模型

- 未配置（缺 `baseUrl/apiKey/model`）
- 加载中（模型响应中）
- 可交互（正常）
- 异常（网络、鉴权、限流、中断、持久化失败）

## 6.2 反馈策略

- 未配置：主区内嵌引导卡，不弹窗打断
- 发送中：显示 assistant 占位行并增量更新
- 失败：消息流内系统提示条（避免全局 toast 轰炸）
- 中断：保留已生成内容，标记“已中断，可继续追问”

## 6.3 危险操作

- 删除会话使用“软确认 + 撤销（2 秒）”
- 若删除当前会话，自动切到最近会话；无会话则新建空会话

## 7. 响应式与可访问性

- 窗口收窄优先折叠次要工具条，再允许侧栏收起
- 焦点态必须可见，支持键盘完整操作
- 文本对比度满足可读性要求

## 8. 开发落地清单

1. 建立 `design-tokens.css`，定义颜色/圆角/阴影/排版变量。  
2. 先搭页面骨架：`Sidebar + Main + Composer`。  
3. 实现会话列表交互态：`default/hover/active/focus`。  
4. 实现混合消息样式与 Markdown 渲染基础样式。  
5. 实现输入框自动增高、发送态、停止生成态。  
6. 接入状态组件：未配置引导、系统提示条、中断态。  
7. 校验动效时长和可访问性（键盘、对比度、焦点）。

## 9. MVP 设计验收标准

- 从空状态到首次发送路径清晰，无阻断。  
- 用户消息与 AI 消息可在 1 秒内视觉区分。  
- 输入框在常见窗口尺寸下不遮挡内容。  
- 出错时用户知道原因与下一步，不依赖控制台。  
- 整体观感为“专业工具”，而非营销化界面。
